# 구현

## 구현이란
  ```
  구현 문제는 "머릿속에 있는 알고리즘을 코드로 나타내 원하는 결과 값을 도출하는것"이다.
  그렇기 때문에 사용하는 언어의 라이브러리를 잘 알지 못하거나 익숙하지 않으면 소스코드 작성에
  애를 먹을 수 있다. 이 유형은 계속 해서 풀어보면서 익숙하지 않았던 라이브러리나 언어 표현 방식에
  대해 익숙해져야 한다.
  ```

 ## 구현의 예
 ```
 구현 문제는 크게 두가지 유형으로 나뉜다. 첫번째는 완전 탐색으로 문제의 모든 경우의 수를 
 구하여 답을 찾는 방식으로 풀어야하고 두번째 시뮬레이션은 문제에서 제시한 알고리즘을 
 한 단계씩 차례대로 풀어나가야 한다.
 
 완전 탐색의 예로 예제 4-2의 시각문제가 있는데 이 문제와 같이 모든 경우의 수를 탐색하여도
 너무 많은 작업을 하지 않아도 될때 완전탐색을 한다.
 
 시뮬레이션의 예로는 예제 4-1의 상하좌우 문제가 있다. 방향을 하나씩 입력받아 받을때 마다 
 이동하는 작업을 한다. 이렇게 주어진 알고리즘을 따라가며 문제를 푸는 방식이 시뮬레이션 방식이다.
 
 ```
